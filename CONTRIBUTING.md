# Contributing Guidelines

Thank you for contributing to Uptime Monitor!

## Workflow Guidelines

1. **Fork the Repository**: Create your own feature branch from `main`.
2. **Coding Standards**:
   - For React code, use functional components and hooks.
   - For Python code, follow PEP 8 standards.
3. **Commit Messages**: Use Conventional Commits standard (e.g. `feat: ...`, `fix: ...`, `docs: ...`).
4. **Testing**:
   - Run `npm run build` in the `frontend` directory before submitting a PR.
   - Run `pytest` in the `backend` directory.

## Pull Request Process

1. Ensure all tests pass.
2. Submit a PR describing your changes and link any related issues.
